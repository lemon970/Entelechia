# Entelechia Image Art Plan

## 范围

本轮只做图片规划，不做生图、不改运行逻辑。规划覆盖：

- 角色形象、角色选择图、头像、地图标记、mod 图标、能量图标。
- 57 张卡牌的独立卡面。
- 18 个 Power 图标。
- 起始遗物和遗物轮廓。
- 现有 `/pic` 素材的复用方式与下一轮制作顺序。

源素材目录：

- `D:\desktop\mod\pic`

游戏内最终资源目录：

- `D:\desktop\mod\Entelechia\Entelechia\images`

代码命名规则：

- 卡牌小图：`Entelechia/images/card_portraits/{id_lower}.png`，尺寸 250 x 190。
- 卡牌大图：`Entelechia/images/card_portraits/big/{id_lower}.png`，尺寸 1000 x 760。
- Power 小图：`Entelechia/images/powers/{id_lower}.png`，尺寸 64 x 64。
- Power 大图：`Entelechia/images/powers/big/{id_lower}.png`，尺寸 256 x 256。
- 遗物小图：`Entelechia/images/relics/{id_lower}.png`，尺寸 94 x 94。
- 遗物大图：`Entelechia/images/relics/big/{id_lower}.png`，尺寸 256 x 256。
- 遗物轮廓：`Entelechia/images/relics/{id_lower}_outline.png`，尺寸 94 x 94。

每张牌必须是不同卡面，不能只做同图换色。可以共享角色形象和机制符号，但每张牌必须至少有一个独立差异点：主体、动作、镜头、场景、道具、特效结构或叙事瞬间。

- 黑发、红眼、尖耳，带轻微非人感。
- 黑色短外套、红色内衬、暗银装饰、哥特礼装轮廓。

## 视觉语言

### 生图参考优先级

1. 身份与服装只参考 `/pic/立绘_隐德来希_1.png`、`立绘_隐德来希_2.png`，不得自行改成长发、礼服裙、盔甲或通用吸血鬼造型。
2. 技能符号优先参考 `/pic` 内原技能图、信物和器皿；萃血应接近“小石榴”式血液结晶与细血线，不画成喷溅血浆。
3. 卡面画法参考《杀戮尖塔 2》实际卡图：粗线、扁平块面、硬边阴影、有限色阶、单一动作和强剪影。只参考画面语言，不复制现成构图或角色。
4. 原设信息以 PRTS 隐德来希条目和官方干员介绍为准；外部同人图不作为身份依据。

### 角色背景锚点

卡面不能只符合黑红外观，还必须按牌义选择至少一个背景锚点：

- 萨卡兹血魔：血液是可精确操纵、结晶和传递信息的媒介，不是失控血浆。
- 玫瑰河畔：她曾从事情报获取、信息清除、人员收揽和暗杀，动作应干净、克制、像完成委托。
- “小石榴”：精巧的血液结晶术法造物，可杀人也可救人，萃血和回流牌优先使用石榴状血晶。
- 坎德勒小姐：她曾以蜡烛匠身份生活，玫瑰蜡烛关联短暂人生、哀悼与信物，不画成邪教火焰。
- 航船与信物：玫瑰河畔成员以航船互称，信物代替自己陪伴珍视之人；终局、余烬和重盈牌可使用河流、航迹、旧信物。
- 血镰协同：绯红壁合是在自己与另一名地面单位处分别召唤血镰进行周期切割，不画成血翼、徽章或两扇门。

背景锚点应藏在动作、道具或场景里，不添加说明文字，也不把每张卡都画成完整剧情插画。

### 反 AI 味约束

所有卡面和图标提示词必须包含：

- 非写实、非厚涂海报；使用手绘卡牌插画、粗粝墨线、平涂色块、硬边阴影和少量纸张纹理。
- 一个主动作、一个机制符号、一个视觉焦点；背景只保留支持动作的两到三层形状。
- 保留不完美的手绘边缘、局部夸张比例和清楚负形，不追求镜面材质与全画面精修。
- 禁止通用手游装备图标、对称徽章、繁复金属包边、镜面塑料、3D 渲染、摄影棚光、过度辉光、粒子堆叠、满屏花瓣。
- 禁止把角色画成正面站立海报；卡面优先使用手、武器、伤口、血线、烛芯或命中瞬间的局部叙事。
- 禁止重新设计脸型、发型和服装；角色出现时必须保持黑色短发、红眼、尖耳、黑红短外套、红色内衬和暗银配件。
- 无文字、无卡框、无水印、无商标。

推荐提示词结构：

`身份锁定 -> 单一动作 -> 机制如何可视化 -> 镜头与剪影 -> 尖塔式手绘画法 -> 有限配色 -> 禁止项`

| 机制 | 视觉符号 | 与其他机制的区分 |
|---|---|---|
| 萃血 | 深红血珠从敌人体内被抽出，沿细线回流角色 | 不是普通伤口流血，而是“血被采集”。 |
| 失血 | 细裂口、冷红血线、滴落、倒计时感 | 比萃血更破碎、更被动，不回流。 |
| 心烛 | 目标体内的心形红烛、白金火、红蜡封印 | 不是普通火球，必须像“生命被点燃”。 |
| 血速 | 红色残影、脉搏环、动脉轨迹、刃轨重复 | 表示额外命中，不只是跑得快。 |
| 自伤 | 割掌、血液主动流入法阵、身体局部红光 | 重点是主动付费，不画成被敌人攻击。 |
| 回复 | 血线逆流、伤口缝合、余烬暖光 | 与萃血可关联，但要有恢复光。 |
| 能量 | 血珠点亮成红白核心、短促脉冲 | 小而亮，避免和心烛火混淆。 |
| 格挡 | 凝血盾、血幕、缝合线、红蜡壁 | 血幕偏薄，血盾偏厚，凝血屏障偏鳞片结构。 |
| 消耗 | 蜡封熔尽、牌页炭化、血契烧毁 | 稀有/升级版如果不消耗，应避免焚毁视觉。 |
| 临时力量 | 红色肌理符文、武器上的短暂血焰 | 不画成永久王冠或长期变身。 |

通用卡面要求：

- 不在画面里写卡名或说明文字。
- 1000 x 760 先画完整画面，再裁切/缩放出 250 x 190。
- 小图必须能看清一个核心动作或核心符号。
- 稀有牌可以更复杂，普通牌保持清晰。
- 每张牌的镜头尽量错开：近景、半身、远景、俯视、侧逆光、道具特写交替使用。

差异化规则：

- 同机制不同图。同为萃血，也要换主体、动作和构图：抽血、凝血、血债、血宴、回流不能共用一张“血滴飞出”图。
- 攻击牌必须有明确武器、动作轨迹或命中瞬间；技能牌必须有明确仪式、代价、资源交换或防御形态；能力牌优先画持续规则或场域。
- 心烛牌统一使用红烛/心脏火芯，但点燃方式必须不同；萃血牌统一使用血珠回流，但抽取媒介必须不同；失血牌统一使用裂口/滴落/枯竭感，但伤口形态必须不同。
- 当前代码按卡牌 id 读取卡面，不按升级状态读取卡面。因此本轮按“每张牌一张正式卡面”规划，不单独规划升级图片文件。若后续要做升级专属卡面，需要先扩展 `EntelechiaCard` 的资源路径规则。

## 卡面差异化检查表

下一轮每张牌生图前都用这张表检查：

- 主体是否不同：角色、敌人、道具、法阵、武器、血珠、烛火至少一种不同。
- 动作是否不同：斩、刺、抽取、缝合、点燃、献祭、重构、复燃不能混用同姿势。
- 镜头是否不同：近景、半身、远景、俯视、仰视、道具特写交替。
- 机制是否可见：玩家不看描述也能猜出萃血/失血/心烛/血速/格挡/自伤中的主机制。
- 稀有度是否拉开：Rare 允许更复杂仪式构图，Basic/Common 更清楚。
- 小图是否可读：缩到 250 x 190 后仍能看清核心符号。
