# 隐德来希全牌库 sim2 平衡验收矩阵

日期：2026-07-10

## 目的

本矩阵用于验证“整体加强、高血过牌与启动、低血续航与爆发”的全牌库调整。心烛理论高值不作为当前削弱理由，但仍检查实现错误、递归结算和意外无限循环。

## 当前 sim2 缺口

- `DamagePerTurnSim` 只记录敌方受到的伤害、能量消耗和心烛账本。
- 木桩每回合恢复到固定生命，玩家不会承受敌方攻击，因此无法验证回复、格挡、死亡率和低血续航。
- `Harness.BeginCombat` 可以设置木桩生命和房间类型，但不能直接设置玩家起始生命。
- `PlayPhase` 有每回合 100 次出牌上限，但 `TrialResult` 不报告是否命中上限。
- `PlayCapture` 已记录抽牌与出牌事件，可以直接派生抽牌数、手动/自动出牌数，无需重新实现事件采集。

## 必须新增的输入

| 输入 | 建议类型 | 用途 |
|---|---|---|
| `StartingPlayerHp` 或 `StartingPlayerHpRatio` | `decimal?` | 覆盖满血、略高半血、恰好半血和 25% 生命 |
| `IncomingDamagePerTurn` | `int` | 在玩家回合结束后施加固定敌方伤害 |
| `ResetDummyEachTurn` | `bool` | 输出木桩沿用现状，生存场景可选择连续战斗 |
| `MaxPlaysPerTurn` | `int`，默认 100 | 保留保护上限并允许专门测试循环 |

固定敌伤必须通过游戏的伤害命令结算，使格挡、受伤钩子、濒死保护和死亡事件正常触发；不能直接用反射改玩家当前生命。

## 必须新增的结果指标

### 每回合

- 玩家回合开始生命。
- 主动失去生命总量。
- 回复总量和有效回复量。
- 获得格挡总量、敌方攻击前格挡、实际承伤。
- 回合结束生命和是否死亡。
- 抽牌数、出牌数、手动出牌数、自动出牌数。
- 起始能量、获得能量、消耗能量、剩余能量。
- 是否命中每回合出牌上限。
- 现有伤害、心烛生成、心烛消耗和剩余池继续保留。

### 每次试验

- 平均每回合伤害、格挡、回复、自伤、抽牌、出牌和能量获得。
- 最终生命、最低生命、死亡回合和死亡率。
- 命中出牌上限的回合数与试验比例。
- 单卡实际打出次数，防止“加入牌组但从未被策略选中”造成假结论。

## 生命档位

以角色当前最大生命为基准，至少测试以下四档：

| 档位 | 50 最大生命示例 | 预期状态 |
|---|---:|---|
| 满血 | 50 | 高血 |
| 略高于半血 | 26 | 高血 |
| 恰好半血 | 25 | 低血 |
| 濒危 | 12 或 13 | 低血 |

恰好半血必须单独列出，专门捕获空收益分支和比较符错误。

## 固定敌伤档位

- `0`：纯资源与输出基线。
- `8`：普通战持续压力。
- `12`：精英级持续压力。
- `16`：濒危保护与大回复压力测试，不作为所有牌组必须存活的常规门槛。

默认使用 `40 seeds`、`5 turns`。单卡行为测试使用固定种子；构筑比较使用全部种子平均值和分位数，不使用只挑最高伤害结果的 Best-of-K 作为生存结论。

## 构筑矩阵

| 构筑 | 主要验证内容 |
|---|---|
| 起始牌组 | 前期下限和基础牌改动是否过强或无效 |
| 高血启动 | 自噬、献血、借血、血色脉冲、血色重构的过牌/加费/压血链 |
| 低血续航 | 血愈、赤色余温、缝合、血债清算、不死血统的回复与格挡 |
| 血速多段 | 低血爆发和多段攻击，不因加强续航而丢失输出特色 |
| 萃血节奏 | 萃血作为回复资源与消耗资源的机会成本 |
| 能量循环 | 血色脉冲、逆流斩、借血、自噬的返能和伪无限风险 |
| 心烛爆发 | 仅做回归，确认非目标改动没有破坏生成、兑现和账本 |
| 惜别引擎 | 长线能力启动速度与稀有牌回报 |

## 单卡边际测试

对每张首批候选牌执行“基准牌组”和“只替换/加入该牌”的同种子对照：

- 血愈：高血比较抽牌与出牌数；低血比较有效回复、最终生命和死亡率。
- 自噬：高血比较获得能量、出牌数和进入低血区的速度。
- 血色重构：分别从满血、26、25、12 生命打出，验证两侧收益和半血无死区。
- 赤色余温：比较本回合未主动失血但已处于低血时的防守收益。
- 借血：比较高血返能稳定性以及低血时是否仍有合理价值。
- 血色脉冲、逆流斩：比较低血行动力，并运行循环检测。
- 血债清算、凝血回流：比较消耗萃血后的回复/过牌收益是否覆盖资源机会成本。
- 不死血统：从 1、4、12 生命测试主动生命代价和敌方致死伤害。

## 验收判据

### 全局

- Mod 与 sim2 均为 `0 warning / 0 error`。
- 现有隐藏测试全部通过；新增阈值、生存和循环测试全部通过。
- 卡牌名称、ID 和图片资源不变。
- 起始牌组当前输出基线 `18.77/turn` 不下降。
- 能量循环当前输出基线 `22.90/turn` 不下降。
- 心烛爆发当前输出基线 `49.47/turn` 不因非心烛改动下降超过 5%。

### 高血阶段

- 加入高血启动牌后，平均抽牌数、获得能量或出牌数至少一项明确提升，且不能只由极少数高随机种子拉高均值。
- 自噬等一次性资源牌必须在前两回合产生可测收益，不能仍然是“支付生命换回一张普通牌的费用”。
- 血色重构从高血打出后必须获得足以覆盖牌费的净资源，并把玩家可靠地送入低血区。

### 低血阶段

- 低血续航构筑在 `IncomingDamagePerTurn = 12` 时，相对改动前降低死亡率，或在死亡率已为 0 时提高平均最终生命与有效格挡。
- 低血续航增强不能把平均伤害压低到改动前的 95% 以下。
- 血愈、赤色余温等低血牌在恰好半血时必须触发完整低血收益。
- 回复统计按有效回复计算，溢出治疗不能制造虚假的续航提升。

### 循环

- 起始牌组和普通/罕见两张牌组合命中 100 次出牌上限的比例必须为 0。
- 明确设计为长线引擎的多组件稀有构筑可以出现伪无限，但必须单独报告组件数量、升级要求、生命消耗和稳定率，不能由一张普通牌或单一低血条件无成本完成。
- 每个返能牌同时报告净手牌变化和净能量变化，避免只看伤害掩盖循环。

## 最小首批测试集

1. 固定种子测试 `26 / 25` 生命的高低血分支边界。
2. 血色重构在 `50 / 26 / 25 / 12` 生命的精确结果测试。
3. 血愈在高血和低血的抽牌、有效回复测试。
4. 自噬的生命支付、能量获得、濒死保护测试。
5. 赤色余温在“低血但本回合未失血”状态下的格挡测试。
6. 血色脉冲与逆流斩的 100 次出牌上限测试。
7. 40 种子、5 回合、12 固定敌伤的低血续航对照矩阵。
8. 现有七套牌组输出回归，心烛爆发只设回归下限，不设削弱目标。
