# 隐德来希心烛第一轮平衡设计

日期：2026-07-10

## 目标

本轮同时修正心烛的两个问题：

1. 生成比例偏低，玩家支付 1 至 2 费后只能获得较小的延迟伤害池。
2. 不同强度的敌人使用相同兑现倍率，心烛在高血量精英和 Boss 战中缺少应有的穿透与爆发感。

心烛仍然是有限伤害池。提高攻击倍率只加快伤害池兑现，不突破已经生成的心烛总量。

## 心烛生成比例

| 卡牌或效果 | 当前 | 调整后 |
|---|---:|---:|
| 烛痕刺 | 2% / 3% | 攻击前生成 5% / 8%，随后造成伤害 |
| 心烛印记 | 4% / 6%，抽 1 | 无心烛时生成 12% / 18% 并返还实际支付的至多 1 点能量；已有心烛时生成 6% / 9%；抽 1，不消耗 |
| 心烛仪式 | 1 费，6% / 9%，消耗 | 0 费，18% / 27%，消耗 |
| 熔血为烛 | 1 费；每 2 点失血转化 1% / 2%，上限 4% / 8% | 0 费；每 2 点失血转化 4% / 6%，上限 16% / 24% |
| 血色横扫 | 2% / 3%，需要本回合失血 | 6% / 9%，触发条件不变 |
| 灵与欲的惜别 | 4% / 6%，群体 | 12% / 18%，群体 |
| 绯红壁合 | 每次萃血触发 1% / 2% | 每次萃血触发 4% / 6% |
| 血魔形态回合播种 | 1% | 每层形态 4% |
| 残烛复燃 | 2 费；恢复已消耗池的 10% / 20%，消耗 | 1 费；恢复合格已消耗池的 50% / 100%，消耗 |

`残烛复燃` 只允许原生心烛伤害增加可复燃额度。复燃产生的心烛再次被消耗时不增加额度，防止多张复燃牌把同一批伤害递归复制。

这些数值以高生命 Boss 为主要校准对象。普通敌人生命较低，生成池仍可能被一次攻击消耗；精英和 Boss 的生成池应能支撑约两至四次普通攻击。生成比例不依靠统一压低，而由消耗、卡位、失血层数消耗或状态前置分别限制。

## 前期启动与纯生成牌费用

起始牌组没有心烛生成牌，因此普通卡必须能够单独启动，而不能要求先凑齐完整组件。

`烛痕刺` 改为先生成心烛再攻击。普通战中它能立即兑现一部分或全部新池；高生命目标会留下剩余池供后续攻击使用。

`心烛印记` 保持 1 费和抽 1，不加入消耗。打出前目标没有心烛时使用 12% / 18% 的首次建池比例，并返还本次实际支付的至多 1 点能量；目标已有心烛时只生成 6% / 9%，且不返能。费用被其他效果降到 0 时不能凭空获得能量。

`心烛仪式` 降为 0 费并维持消耗。它不抽牌、不直接造成伤害，主要成本是卡位和一次性使用。

`熔血为烛` 降为 0 费。它必须先取得失血层数并消耗最多 8 层失血，资源消耗已经构成实际费用。

`残烛复燃` 降为 1 费并维持消耗。恢复公式为：

```text
min(
  ceil(原生已消耗心烛 * 50% / 100%) - 累计已复燃量,
  目标当前生命 - 当前心烛池
)
```

账本分别记录：

```text
原生心烛已消耗量
尚未消耗的复燃心烛量
累计已复燃量
```

消耗复燃产生的心烛时，优先扣除“尚未消耗的复燃心烛量”，这部分不加入原生已消耗量。这样基础版最多重放原生已消耗池的 50%，升级版最多完整重放一次，不能递归增长。

## 敌人等级兑现倍率

通过 `HeartCandlePower.Owner.CombatState.Encounter.RoomType` 判断当前战斗等级。

| `RoomType` | 基础兑现倍率 |
|---|---:|
| `Monster` | 2.0 倍 |
| `Elite` | 2.5 倍 |
| `Boss` | 3.0 倍 |
| 其他或无法判断 | 2.0 倍 |

每次攻击触发的心烛伤害仍按以下公式计算：

```text
min(
  ceil(攻击未格挡伤害 * 最终兑现倍率),
  心烛剩余池量,
  目标当前生命
)
```

心烛伤害继续使用失去生命效果，不受格挡影响，也不能再次触发心烛或萃血递归。

## 状态增量

| 条件 | 兑现倍率增量 |
|---|---:|
| 每层血魔形态 | +0.5 |
| 使用红烛万象触发 | +0.75 |

所有增量做加法，不做乘法，也不设置倍率硬上限。最终倍率为：

```text
敌人等级基础倍率
+ 血魔形态层数 * 0.5
+ 当前攻击由红烛万象触发时的 0.75
```

例如，一层血魔形态下使用红烛万象攻击 Boss，倍率为 `3.0 + 0.5 + 0.75 = 4.25`；两层血魔形态下为 `4.75`。心烛有限池仍然限制总伤害，因此多层形态主要提高大池的兑现速度。

`BloodDemonFormPower` 从 `Single` 改为可计数叠加。每张血魔形态都增加一层，现有按 `Amount` 获得力量的逻辑继续生效；回合播种量和本回合首次失血获得的血速也按层数计算。

多张心烛生成牌继续把各自按目标当前生命计算出的池量相加。红烛万象是单次攻击来源，不保留永久层数；多张红烛万象分别在各自打出时提供 `+0.75`。

低生命倍率暂不加入本轮。低生命收益将与 `血愈`、返能和过牌调整一起进入独立测试，避免在同一轮同时改变生成量、敌人倍率和低生命倍率，导致无法判断收益来源。

## 实现边界

- 不修改任何卡牌名称或 ID。
- 不改变心烛按目标当前生命百分比生成的规则。
- 不改变心烛有限池和消耗逻辑。
- 不把敌人具体类名写入白名单。
- 召唤物和特殊战斗沿用当前遭遇的 `RoomType`。
- 模拟器没有有效遭遇等级时按普通敌人处理。
- 烛痕刺在攻击前生成心烛。
- 心烛印记不消耗；首次建池只返还本次实际支付的至多 1 点能量。
- 心烛仪式和熔血为烛降为 0 费；残烛复燃降为 1 费。
- 残烛复燃只按原生已消耗心烛计算额度，复燃池再次消耗不产生新额度。
- 当前心烛池与本次复燃量之和不超过目标当前生命。
- 血魔形态层数、回合播种、力量和首次失血血速按相同 `Amount` 结算。
- 同步更新中英文卡牌和能力描述中涉及的百分比。

## 测试要求

### 行为测试

增加或更新 sim2 烟雾测试，覆盖：

1. 普通战斗使用 2.0 倍。
2. 精英战斗使用 2.5 倍。
3. Boss 战斗使用 3.0 倍。
4. 未知战斗类型回退到 2.0 倍。
5. 每层血魔形态增加 0.5。
6. 红烛万象只增加 0.75。
7. 多层血魔形态与红烛万象按加法同时生效。
8. 心烛池不足时不会突破剩余池量。
9. 心烛伤害不会触发递归。
10. 八项生成比例与升级比例符合上表。
11. 多张生成牌把各自基于目标当前生命计算的池量相加。
12. 烛痕刺先生成再攻击，并能由自身攻击立即兑现心烛。
13. 心烛印记不消耗；首次建池返还实际支付的能量，0 费打出时不额外产能。
14. 心烛仪式和熔血为烛为 0 费，残烛复燃为 1 费。
15. 残烛复燃分别恢复原生已消耗池的 50% / 100%，并受目标当前生命上限约束。
16. 消耗复燃池不会增加新的可复燃额度，多张复燃牌不能递归复制同一批池量。
17. 血魔形态的力量、播种、血速和兑现增量按层数结算。

观察值至少包括总攻击伤害、心烛生成量、心烛消耗量、战斗结束时未消耗心烛和每点能量造成的总伤害。

## 验收标准

- 心烛组件在 `K=5` 和 `K=10` 下能明显高于无心烛的普通攻击牌组，而不是只在 `K=40` 理想顺序下爆发。
- 普通、精英、Boss 的单次兑现效率严格递增。
- 高倍率不能突破心烛池总量。
- 普通敌人不会因为生成比例和兑现倍率同时提高而普遍被一张普通心烛牌直接击杀。
- 精英战中的主要生成牌至少支持约两次普通攻击兑现。
- Boss 战中的高投入生成牌至少支持约三次普通攻击兑现，升级后可接近四次。
- 起始牌组只加入烛痕刺或心烛印记之一时，就能在 3 点能量内完成建池与至少一次攻击兑现。
- 纯生成牌不会因为费用占满回合而阻止后续攻击，且心烛印记不能在 0 费打出时产生净能量。
- 残烛复燃基础版最多恢复原生已消耗池的一半，升级版最多恢复全部，任何数量的复燃牌都不能递归扩大同一批额度。
- 所有既有 `hidden-tests` 继续通过。
